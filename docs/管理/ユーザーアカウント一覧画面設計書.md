# 管理画面 ユーザーアカウント一覧画面 設計書

---

## 画面名

* 画面ID：CMS-USER-LIST-001
* 画面名：ユーザーアカウント一覧
* 種別：Web（管理画面）
* 対象：管理者 / 運営スタッフ
* 目的：
  推しドラに登録されているユーザーアカウントを一覧で確認し、
  状態確認・詳細確認・制御を行う

---

## 画面位置づけ

* 管理画面ダッシュボードから遷移
* サイドメニュー「ユーザー管理」の起点画面

---

## 画面遷移

### 遷移元

* ダッシュボード（CMS-DASHBOARD-001）
* サイドメニュー「ユーザー管理」

### 遷移先

* ユーザー詳細画面（CMS-USER-DETAIL-001）
* ユーザー利用履歴画面（任意・将来）
* 強制退会／停止確認モーダル

---

## 画面構成（全体）

```
[ ヘッダー ]
 ├ ロゴ
 ├ 管理者名
 └ ログアウト

[ サイドメニュー ]
 ├ ダッシュボード
 ├ 動画管理
 ├ キャスト・スタッフ管理
 ├ コイン管理
 ├ ユーザー管理（選択中）
 └ 設定

[ メインコンテンツ ]
 ├ 検索・絞り込みエリア
 ├ ユーザー一覧テーブル
 └ ページネーション
```

---

## 検索・絞り込みエリア

### 目的

ユーザー数増加を想定し、対象ユーザーを素早く特定する

### 絞り込み項目

| 項目          | 種別          | 内容          |
| ----------- | ----------- | ----------- |
| ユーザーID      | テキスト        | 完全一致        |
| 表示名（ニックネーム） | テキスト        | 部分一致        |
| メールアドレス     | テキスト        | 部分一致        |
| 電話番号        | テキスト        | 部分一致        |
| 登録日         | 日付From / To | 期間指定        |
| アカウント状態     | セレクト        | 有効 / 停止     |
| 年齢区分        | セレクト        | 年齢制限判定用（任意） |

### 操作

* 検索
* リセット

---

## ユーザー一覧テーブル

### 表示項目

| 列名       | 内容      | 表示仕様       |
| -------- | ------- | ---------- |
| ユーザーID   | システムID  | コピー可能      |
| プロフィール画像 | アイコン    | 小サイズ       |
| 表示名      | ニックネーム  | 1行表示       |
| メールアドレス  | 登録メール   | 省略表示可      |
| 電話番号     | 登録電話    | マスク表示      |
| 登録日      | 登録日時    | YYYY/MM/DD |
| コイン残高    | 保有コイン   | 数値         |
| アカウント状態  | 有効 / 停止 | バッジ表示      |
| 操作       | 詳細 / 停止 | ボタン        |

---

## 操作ボタン詳細

### 詳細

* ユーザー詳細画面へ遷移
* 個人情報、購入履歴、視聴履歴、コメント履歴を確認可能

### 停止 / 再開

* 押下時に確認モーダル表示

  * 文言例：

    * `このユーザーを停止しますか？`
* 実行後、即時状態反映

※ 削除ではなく **論理停止** を基本とする

---

## ページネーション

* 1ページあたり：20件
* 前へ / 次へ
* ページ番号指定

---

## 状態表示仕様

### アカウント状態

| 状態 | 表示     | 挙動     |
| -- | ------ | ------ |
| 有効 | 通常表示   | 操作可    |
| 停止 | グレーアウト | ログイン不可 |

---

## エラーハンドリング

* 一覧取得失敗
  → `ユーザー一覧を取得できませんでした`
* 状態変更失敗
  → トースト通知でエラー表示

---

## 権限制御

| 操作    | 管理者 | 運営 |
| ----- | --- | -- |
| 一覧閲覧  | ○   | ○  |
| 詳細閲覧  | ○   | ○  |
| 停止／再開 | ○   | ×  |
| 完全削除  | ×   | ×  |

---

## 非機能要件

* 初期表示：2秒以内
* 非同期検索
* 大量ユーザーでも安定表示

---

## 備考

* 個人情報は一覧では最小限表示（マスク）
* 詳細画面でのみ全情報表示
* ユーザーの違反対応・問い合わせ対応の起点として利用
