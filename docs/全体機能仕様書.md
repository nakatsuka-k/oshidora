# 推しドラ 機能一覧（アプリ＋CMS統合版）

## フェーズ定義

* **PH1（初期）**：運営主導でコンテンツ供給・安全運用を最優先
* **PH2（拡張）**：レコメンド強化／コメントリアクション／換金拡張など

---

## 1) アプリ：基本機能（起動・認証・課金）

* **アプリ起動時ショート動画表示**
* **ログイン機能**

  * 初回ログイン時：ユーザー向けチュートリアル（ウォークスルー）を実施
* **会員登録機能**
* **二段階認証**

  * メール認証＋SMS認証（SMS認証を実装）
* **ウォークスルー（初回チュートリアル）**
* **課金システム（Stripe）**

  * サブスク加入（動画視聴権）
  * コイン購入（推しポイント付与用）

---

## 2) アプリ：トップページ

* **検索機能**

  * キーワード検索

* **ピックアップ動画一覧表示**

  * 手動で表示動画を選択（=CMSで枠を設定）

* **おすすめ動画一覧表示**

  * 手動で表示動画を選択（=CMSで枠を設定）

* **ランキング表示**

  * 再生数等から算出、日次更新

* **PH2：おすすめ（レコメンド）**

  * 閲覧した動画に対するレコメンド動画表示

---

## 3) アプリ：MYページ

* MYページトップ
* **スタッフ・キャストお気に入り一覧**

  * お気に入りしたキャスト/スタッフの新着動画・お知らせが確認できる
* **イチオシ設定機能**

  * キャスト/スタッフを「イチオシ」設定
  * 推しコメント記載（プロフィールから閲覧可能）

### コイン

* 無料コイン管理
* 保有コイン表示
* コイン購入（Stripe）
* コイン履歴表示（購入/付与履歴）
* **PH2：コイン換金（ギフトカード交換）**

  * paypayポイント以外への換金
  * paypayポイントに付与されたコインを換金

### その他

* 動画閲覧ヒストリー
* アカウント編集
* パスワード変更
* 通知設定編集

  * 「大切なお知らせ（常時ON）」＋「お気に入り公開・ランキング更新（OFF可）」
  * プッシュ通知：手動（即時/予約）
  * メール：プレーンテキスト（内容によりHTML切替）
  * プッシュ通知タップ時：アプリ起動 or LP遷移

    * LP：トップ画像＋テキストテンプレ表示
* プロフィール登録（AI半自動）

  * 画像以外はAI読み取り→仮入力→保存前に手動修正可
* **プロフィール編集**

  * 編集後、反映前に運営承認が必要（=CMS「プロフィール承認」へ）

---

## 4) アプリ：動画（視聴・作品情報）

* 動画一覧（検索結果）
* 動画詳細
* 動画再生
* 自動エピソード更新
* 字幕ON/OFF

  * 字幕あり/なしを別MP4で登録し、視聴時選択
  * 評価などは同一作品として統一
* 縦/横再生対応
* サムネイル登録（画像アップロード。動画キャプチャ作成はしない）
* エピソード一覧（関連動画紐付け再生）
* 画面録画防止（スクショ時に黒塗り等でキャプチャ不可）
* ダウンロード不可

# 8) CMS（管理画面）機能（アプリ要件を反映して運用仕様を上書き）

※画面IDはPDF準拠 

## CMSの前提

* **運営のみ利用**（少なくともPH1）
* 将来：運営 / 事務所 / 演者で権限分離 

---

## 8-1. 認証・ダッシュボード

* **CMS_00001** ログイン（メール+PW） 
* **CMS_00002** ホーム/ダッシュボード 
  追加で反映（今回の要件）：

  * **未対応コメント件数**を表示（要件）
  * 配信予定動画 / 承認待ち（動画・プロフィール・コメント）を集約

---

## 8-2. コメント承認（運用仕様を確定）

* **CMS_00007** 未承認コメント一覧 
* **CMS_00008** コメント詳細（承認/否認） 
* **CMS_00028** コメント一覧（動画/演者など） 
* **CMS_00029** コメント編集（削除等） 

今回の追加要件（ここが“未反映”だった部分）：

* コメント管理ステータスは **3つ**

  * **未対応非公開**
  * **対応済み非公開**
  * **公開済み**
* **既読管理はしない**
* **新規コメントがあった場合、メール通知**
* ダッシュボードに **未対応コメント件数**

---

## 8-3. 動画管理（字幕2MP4・エピソード・ピックアップ/おすすめ連動）

* **CMS_00024** 動画一覧 
* **CMS_00025** 動画詳細・編集（字幕あり/なしMP4登録、公開制御） 
* **CMS_00030** 動画アップロード 
* **CMS_00037/38** ピックアップ設定 
* **CMS_00026/27** おすすめ設定 

反映ポイント（アプリ要件との接続）：

* アプリTopの「ピックアップ」「おすすめ（手動）」はCMSで枠管理
* 字幕ON/OFFは **別MP4を同一作品に紐付ける**運用（評価統一）
* エピソード一覧は **シリーズ紐付け**で実現（CMS_00025で登録）

---

## 8-4. プロフィール承認（アプリ編集→運営承認）

* あなたの要件：「プロフィール編集は反映前に運営承認」
* CMS側の置き場：

  * **ユーザー詳細（CMS_00043/44周辺）**＋「プロフィール承認」運用
    （PDFでは“運営が編集可能か要検討”の記述があるため、ここを**承認フローとして確定**させるのが自然）

# ステータス表（運用・実装でそのまま使える定義）

## A. コメントステータス（確定：3状態）

| ステータス   | 表示先     | 意味                   | 遷移                 |
| ------- | ------- | -------------------- | ------------------ |
| 未対応非公開  | CMSのみ   | 未審査。アプリ非表示。          | → 公開済み / → 対応済み非公開 |
| 公開済み    | アプリ＋CMS | 審査OK。公開中。            | → 対応済み非公開（取り下げ）    |
| 対応済み非公開 | CMSのみ   | 問題ありで非公開（削除/否認含む扱い）。 | → 公開済み（再公開するなら）    |

**通知ルール（要件）**

* 新規コメント作成時：運営にメール通知
* 既読管理：なし
* ダッシュボード：未対応件数表示

---

## B. プロフィール変更ステータス（承認制）

| ステータス     | 表示先           | 意味          | 遷移            |
| --------- | ------------- | ----------- | ------------- |
| 反映済み      | アプリ＋CMS       | 現在有効なプロフィール | → 申請中         |
| 申請中（承認待ち） | CMS中心         | ユーザー編集後、未反映 | → 承認 / → 差し戻し |
| 差し戻し（否認）  | CMS＋（アプリ任意通知） | 修正要。理由を保持。  | → 再申請         |
| 承認        | システム内部        | 承認時に反映済みに置換 | → 反映済み        |

※「差し戻し理由コメント」「通知するか」は運用で決め打ち推奨

---

## C. 動画公開ステータス（最低限）

| ステータス | 表示先              | 意味      | 備考              |
| ----- | ---------------- | ------- | --------------- |
| 非公開   | CMSのみ            | 視聴不可    | 初期はこれ＋公開のみでもOK  |
| 公開    | アプリ＋CMS          | 視聴可     | ランキング集計対象       |
| 配信予約  | CMS＋（アプリ側は日時で表示） | 指定日時に公開 | CMS_00003/04の枠  |

**将来拡張するなら**

* 承認待ち / 否認 など（事務所投稿を始めるフェーズで追加）

---

## D. 通知配信ステータス（お知らせ/プッシュ）

| ステータス | 意味                 |
| ----- | ------------------ |
| 下書き   | 作成中・未配信            |
| 予約    | 指定日時に送信予定          |
| 送信済み  | プッシュ送信済み（お知らせ掲載済み） |
| 取消    | 予約を取り消した状態         |

---

## E. コイン取引ステータス（履歴用）

| ステータス | 意味             |
| ----- | -------------- |
| 成功    | 購入/付与完了        |
| 失敗    | 決済失敗・付与失敗      |
| 取消/返金 | 返金/取消で差分処理が必要  |
| 保留    | ストア確認待ち等（必要なら） |
